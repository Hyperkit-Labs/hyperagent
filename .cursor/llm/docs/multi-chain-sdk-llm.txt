# Multi-Chain SDK Integration Guide for HyperAgent

Integration guide for Avalanche, BNB Chain, Filecoin (FVM), and SKALE. Aligns with the chain registry (`infra/registries/network/chains.yaml`) and the deploy pipeline (RPC, chainId, explorer verification). For AI agents (ROMA/HyperAgent): use as implementation guidance; favor clarity and determinism.

## Unified deployment abstraction

Use a single entry point that dispatches to chain-specific logic. RPC URL, chainId, and explorer URL MUST come from the chain registry (or env override) so one source of truth drives compile, deploy, and verify.

```ts
type DeployParams = {
  chain: 'avalanche' | 'bnb' | 'filecoin' | 'skale';
  templateId: 'erc20' | 'erc721' | 'upgradeable';
  params: Record<string, unknown>;
};

type DeployResult = {
  chainId: number;
  address: string;
  txHash: string;
  verified: boolean;
};

async function deployContract(input: DeployParams): Promise<DeployResult> {
  const registry = await getChainFromRegistry(input.chain); // from infra/registries/network/chains.yaml
  const { rpcUrl, chainId, explorerUrl } = registry;
  switch (input.chain) {
    case 'avalanche':
      return deployAvalanche(registry, input);
    case 'bnb':
      return deployBnb(registry, input);
    case 'filecoin':
      return deployFilecoin(registry, input);
    case 'skale':
      return deploySkale(registry, input);
    default:
      throw new Error(`Unsupported chain: ${input.chain}`);
  }
}
```

Where each chain's native SDK is used:
- Avalanche: CLI (`avalanche blockchain deploy`) for subnet/L1 lifecycle; for C-Chain contract deploy use ethers/viem + RPC. Explorer verification via Snowtrace API (API key often required).
- BNB: Standard EVM path; ethers/viem + BSC RPC. Verification via BSCScan API (API key required).
- Filecoin: FVM is EVM-compatible; use ethers/viem with FVM RPC. Address formatting (f4 vs 0x) may need FVM helpers; verification via Filfox/Blockscout if documented.
- SKALE: ethers/viem + SKALE RPC; use `--legacy` for Foundry/Hardhat where required. Verification via Blockscout (verifier-url per chain); no API key for Blockscout on SKALE.

## Avalanche

### Documentation

- Builder hub: https://build.avax.network
- CLI commands: https://build.avax.network/docs/tooling/cli-commands
- RPC reference: https://build.avax.network/docs/rpcs/c-chain
- Developer tools: https://build.avax.network/docs/tooling

### Install & setup

```bash
pnpm add ethers viem
# Optional: Avalanche CLI for subnet/L1 lifecycle (not for C-Chain contract deploy)
# curl -sSfL https://raw.githubusercontent.com/ava-labs/avalanche-cli/main/scripts/install.sh | sh
```

Configure from chain registry: chainId 43114 (C-Chain mainnet), RPC `https://api.avax.network/ext/bc/C/rpc`. Fuji testnet has separate RPC. No extra system deps for C-Chain deploy with ethers/viem.

### Basic deployment code

```ts
import { ethers } from 'ethers';
import { avalancheConfig } from './chains/avalanche'; // or from registry

async function deployAvalancheERC20() {
  const provider = new ethers.JsonRpcProvider(avalancheConfig.rpcUrl);
  const wallet = new ethers.Wallet(process.env.DEPLOYER_PRIVATE_KEY!, provider);
  const factory = new ethers.ContractFactory(abi, bytecode, wallet);
  const gasEstimate = await factory.deploy.estimateGas('Token', 'TKN', 18);
  const contract = await factory.deploy('Token', 'TKN', 18, { gasLimit: gasEstimate });
  await contract.waitForDeployment();
  console.log('Avalanche ERC20 deployed at', await contract.getAddress());
  console.log('tx hash', contract.deploymentTransaction()?.hash);
}
```

Use chain registry for `avalancheConfig.rpcUrl` and `chainId` (43114 for C-Chain).

### Explorer verification

- Snowtrace (C-Chain): https://snowtrace.io. Verification via Snowtrace API; API key typically required. Use Hardhat verify plugin or Foundry verify with `--verifier-url` for Snowtrace API endpoint.
- Do not spam `eth_estimateGas`; cache gas estimates when batching.

### Where to hook into deployContract()

- Compile: use existing compile service (Hardhat/Foundry); no Avalanche-specific compile step for C-Chain.
- Deploy: use ethers/viem with registry RPC and chainId.
- Verify: call Snowtrace verification API (or Hardhat/Foundry verify) with explorer API URL from registry; pass API key from env.

### Gotchas & limits

- C-Chain is EVM-compatible; no special ABI or opcodes for standard ERC20/721.
- Avalanche CLI is for subnet/L1 create/deploy/configure, not for deploying a single contract on C-Chain.
- Rate limits on public RPC; use a dedicated RPC provider for production.
- Fuji testnet: use Fuji RPC and faucet for AVAX; mainnet and testnet have different chain IDs.

## BNB Chain

### Documentation

- Docs: https://docs.bnbchain.org
- BSC quick guide: https://docs.bnbchain.org/bnb-smart-chain/developers/quick-guide
- JSON-RPC: https://docs.bnbchain.org/bnb-smart-chain/json_rpc/json-rpc-endpoint/
- Wallet & RPC: https://docs.bnbchain.org/bnb-smart-chain/wallet-configuration/

### Install & setup

```bash
pnpm add ethers viem
```

BSC is EVM-compatible. Use standard Ethereum SDKs (ethers, viem, web3.js). RPC: e.g. `https://bsc-dataseed.bnbchain.org` (mainnet). Chain ID 56 (BSC mainnet). Configure RPC and chainId from chain registry.

### Basic deployment code

```ts
import { ethers } from 'ethers';

async function deployBnbERC20() {
  const rpcUrl = process.env.BNB_RPC_URL ?? 'https://bsc-dataseed.bnbchain.org';
  const provider = new ethers.JsonRpcProvider(rpcUrl);
  const wallet = new ethers.Wallet(process.env.DEPLOYER_PRIVATE_KEY!, provider);
  const factory = new ethers.ContractFactory(abi, bytecode, wallet);
  const gasEstimate = await factory.deploy.estimateGas('Token', 'TKN', 18);
  const contract = await factory.deploy('Token', 'TKN', 18, { gasLimit: gasEstimate });
  await contract.waitForDeployment();
  console.log('BSC ERC20 deployed at', await contract.getAddress());
  console.log('tx hash', contract.deploymentTransaction()?.hash);
}
```

Use registry for `rpcUrl` and chainId (56).

### Explorer verification

- BSCScan: https://bscscan.com. Verification requires BSCScan API key. Use Hardhat `@nomicfoundation/hardhat-verify` or Foundry `forge verify-contract` with BSCScan verifier and API key in env.

### Where to hook into deployContract()

- Compile: existing compile service; no BNB-specific step.
- Deploy: ethers/viem + registry RPC and chainId.
- Verify: BSCScan API with API key from env; use standard Etherscan-compatible verification flow.

### Gotchas & limits

- BSC uses BNB for gas; ensure deployer has BNB. Testnet faucet: https://docs.bnbchain.org/bnb-smart-chain/faucet/
- Rate limits on public RPC; use a provider (e.g. NodeReal, Ankr) for production.
- opBNB (L2) has different chain ID (204); use correct RPC and chainId from registry per chain.

## Filecoin (FVM)

### Documentation

- Filecoin docs: https://docs.filecoin.io
- Smart contracts: https://docs.filecoin.io/smart-contracts/
- FVM is EVM-compatible; use standard EVM tooling and RPC endpoints where documented.

### Install & setup

```bash
pnpm add ethers viem
```

FVM exposes EVM-style RPC. Chain ID 314 (Filecoin mainnet). RPC URL from registry or Filecoin docs. Address format: FVM supports 0x-style addresses for EVM actors; f4 (delegated) addresses may require conversion helpers for display or indexing.

### Basic deployment code

```ts
import { ethers } from 'ethers';

async function deployFilecoinERC20() {
  const rpcUrl = process.env.FILECOIN_RPC_URL ?? 'https://api.node.glif.io/rpc/v1';
  const provider = new ethers.JsonRpcProvider(rpcUrl);
  const wallet = new ethers.Wallet(process.env.DEPLOYER_PRIVATE_KEY!, provider);
  const factory = new ethers.ContractFactory(abi, bytecode, wallet);
  const gasEstimate = await factory.deploy.estimateGas('Token', 'TKN', 18);
  const contract = await factory.deploy('Token', 'TKN', 18, { gasLimit: gasEstimate });
  await contract.waitForDeployment();
  console.log('Filecoin FVM ERC20 deployed at', await contract.getAddress());
  console.log('tx hash', contract.deploymentTransaction()?.hash);
}
```

Use registry for `rpcUrl` and chainId (314).

### Explorer verification

- Filfox and other Filecoin explorers may support contract verification; check current FVM verification docs. If Blockscout/Etherscan-style API exists, use same pattern as other chains (verifier URL + optional API key).

### Where to hook into deployContract()

- Compile: existing compile service; FVM is EVM-compatible.
- Deploy: ethers/viem + FVM RPC and chainId from registry.
- Verify: use FVM explorer verification API if documented; otherwise mark `verified: false` and document manual verification steps.

### Gotchas & limits

- Filecoin mainnet uses FIL for gas; ensure deployer has FIL. Testnets may have faucets.
- FVM and storage deal flows are separate; this guide covers FVM contract deploy only.
- Address handling: prefer 0x for EVM contract addresses; convert to f4 only when required by another system.

## SKALE

### Documentation

- Docs index: https://docs.skale.space/llms.txt
- Connect: https://docs.skale.space/developers/integrate-skale/connect-to-skale.md
- Deploy ERC-20: https://docs.skale.space/cookbook/smart-contracts/deploy-erc20-token.md
- Verify: https://docs.skale.space/cookbook/smart-contracts/verify-smart-contracts.md
- JSON-RPC: https://docs.skale.space/developers/integrate-skale/json-rpc-api.md

### Install & setup

```bash
pnpm add ethers viem
# Foundry (optional): forge + cast for deploy/verify
# Hardhat: npx hardhat (use --legacy for tx type where needed)
```

SKALE Base Mainnet: RPC `https://skale-base.skalenodes.com/v1/base`, chainId 1187947933. SKALE Base Sepolia: RPC `https://base-sepolia-testnet.skalenodes.com/v1/jubilant-horrible-ancha`, chainId 324705682. Use chain registry; no API key for RPC. sFUEL/faucet for testnet: https://base-sepolia-faucet.skale.space (Sepolia).

### Basic deployment code

```ts
import { ethers } from 'ethers';

async function deploySkaleERC20() {
  const rpcUrl = process.env.SKALE_RPC_URL ?? 'https://base-sepolia-testnet.skalenodes.com/v1/jubilant-horrible-ancha';
  const chainId = 324705682; // SKALE Base Sepolia; use registry
  const provider = new ethers.JsonRpcProvider(rpcUrl);
  const wallet = new ethers.Wallet(process.env.DEPLOYER_PRIVATE_KEY!, provider);
  const factory = new ethers.ContractFactory(abi, bytecode, wallet);
  // SKALE: zero gas fee but still need sFUEL; gasLimit can be set
  const gasEstimate = await factory.deploy.estimateGas('Token', 'TKN', 18);
  const contract = await factory.deploy('Token', 'TKN', 18, { gasLimit: gasEstimate });
  await contract.waitForDeployment();
  console.log('SKALE ERC20 deployed at', await contract.getAddress());
  console.log('tx hash', contract.deploymentTransaction()?.hash);
}
```

Use registry for `rpcUrl`, `chainId`, and `explorerUrl`.

### Explorer verification

- Each SKALE chain has a Blockscout explorer. API URL format: testnet `https://<chain>.explorer.testnet.skalenodes.com/api`, mainnet `https://<chain>.explorer.mainnet.skalenodes.com/api`. Example (Sepolia): `https://base-sepolia-testnet-explorer.skalenodes.com/` (browser); API for verify: use Blockscout API URL for that chain.
- Foundry: `forge verify-contract --rpc-url <rpc> <address> src/Contract.sol:Contract --verifier blockscout --verifier-url <explorer_api_url>`.
- Hardhat: `@nomicfoundation/hardhat-verify` with `etherscan.customChains` pointing to the chain's Blockscout API URL. API key not required for Blockscout on SKALE.

### Where to hook into deployContract()

- Compile: existing compile service; SKALE is EVM-compatible.
- Deploy: ethers/viem + registry RPC and chainId. If using Foundry/Hardhat scripts, use `--legacy` (Foundry) or equivalent for SKALE when required.
- Verify: call Blockscout verification (forge verify-contract or Hardhat verify) with `--verifier-url` from registry (explorer API URL). Map chain slug to explorer API URL from registry or SKALE Portal.

### Gotchas & limits

- Use `--legacy` for Foundry deploy on SKALE when the chain requires it.
- SKALE has zero gas fees but sFUEL is still required for sending transactions; get sFUEL from faucet for testnet.
- Explorer API URL is per-chain; do not hardcode one chain's URL for all SKALE chains. Use registry or SKALE Portal to resolve explorer API URL by chainId/slug.
- Do not spam RPC; use normal backoff and rate limiting.

## Chain registry alignment

- Single source: `infra/registries/network/chains.yaml`. Each chain has `id` (chainId), `chainlist.rpcUrls[0]`, `chainlist.blockExplorerUrls[0]`, `hyperagent.slug`.
- Deploy pipeline must read RPC URL, chainId, and explorer URL from this registry (or from API that serves it). No hardcoded RPC/explorer per chain in application code.
- For verification: derive explorer API URL from blockExplorerUrls (e.g. append `/api` for Blockscout) or maintain a small map slug -> verification API URL in config if explorer format differs.
