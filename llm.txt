# HyperKit Agent - Technology Stack & Integration Guide

## Core Technology Stack

### Smart Contract Development
- **OpenZeppelin**: Security-focused smart contract libraries and standards
- **Foundry**: Fast, portable toolkit for Ethereum application development
  - Used for: Solidity compilation, testing, deployment
- **Hardhat**: Ethereum development environment
  - Used for: Development workflow, testing, deployment scripts

### Blockchain Infrastructure
- **EigenDA**: Data availability layer for Mantle mainnet
  - Used for: Batch transaction data availability on Mantle
- **IPFS/Pinata**: Decentralized storage and content addressing
  - Gateway: `https://gateway.pinata.cloud`
  - Used for: Contract metadata, artifact storage, decentralized file hosting
- **libp2p**: Modular networking stack for peer-to-peer applications
  - Used for: P2P networking, content routing

### Payment & Wallet Infrastructure
- **Thirdweb x402 SDK**: Open-source protocol for HTTP 402 Payment Required
  - Client SDK: `thirdweb/x402` for React/TypeScript
  - Server SDK: `thirdweb/x402` facilitator for payment settlement
  - Payment tokens: ERC-2612 permit or ERC-3009 sign with authorization (USDC)
  - Supported chains: 170+ EVM chains
  - References:
    - https://portal.thirdweb.com/x402
    - Client: `useFetchWithPayment` hook for automatic 402 handling
    - Server: `settlePayment` function with facilitator
- **Thirdweb Wallets SDK**: User wallet creation and management
  - Authentication methods: Email, SMS, OAuth, Passkeys, SIWE
  - Server wallets: For backend operations
  - References:
    - https://portal.thirdweb.com/wallets
    - API: `/v1/auth/initiate`, `/v1/auth/complete`
    - Social auth: `/v1/auth/social`
- **Thirdweb Gas Sponsorship**: Sponsor gas fees for transactions
  - Supports both EIP-7702 (recommended) and ERC-4337
  - EIP-7702: Default execution mode for user and server wallets
  - ERC-4337: Account abstraction standard for gasless transactions
  - References:
    - https://portal.thirdweb.com/wallets/sponsor-gas
    - API: `/v1/contracts/write` with automatic gas sponsorship
    - Sponsorship policies: Configure rules for gas sponsorship

### Database & Backend
- **Supabase**: PostgreSQL database with pgbouncer connection pooling
  - Connection format: `postgresql://postgres:PASSWORD@db.PROJECT_REF.supabase.co:5432/postgres`
  - Pooling: Uses pgbouncer (transaction mode)
  - Python: Uses `psycopg` (async) for pgbouncer compatibility
  - TypeScript: Uses `pg` Pool with reduced max connections (5) for Supabase
  - Schema: `hyperagent` namespace for all tables

### Frontend Stack
- **Next.js**: React framework with App Router
- **Tailwind CSS**: Utility-first CSS framework
- **TypeScript**: Type-safe JavaScript

  - Cross-chain messaging: L1 ↔ L2 contract communication

  - References:

## Integration Patterns

### Thirdweb x402 Payment Flow

#### Client Side (React/Next.js)
```typescript
import { useFetchWithPayment } from "thirdweb/react";
import { createThirdwebClient } from "thirdweb";

const client = createThirdwebClient({ clientId: "your-client-id" });

// Automatic 402 handling with wallet popup
const { fetchWithPayment, isPending } = useFetchWithPayment(client);

const response = await fetchWithPayment("https://api.example.com/paid-endpoint");
```

#### Server Side (TypeScript API)
```typescript
import { settlePayment } from "thirdweb/x402";
import { facilitator } from "thirdweb/x402";

const thirdwebX402Facilitator = facilitator({
  client,
  serverWalletAddress: "0xYourWalletAddress",
});

const result = await settlePayment({
  resourceUrl: "https://api.example.com/premium-content",
  method: "GET",
  paymentData: request.headers.get("PAYMENT-SIGNATURE"),
  payTo: "0xYourWalletAddress",
  network: "avalanche_fuji",
  price: "$0.01",
  facilitator: thirdwebX402Facilitator,
});
```

#### Payment Headers
- **v1**: `X-PAYMENT` header
- **v2**: `PAYMENT-SIGNATURE` header (preferred)

### Supabase Database Configuration

#### Python (FastAPI)
```python
# Uses psycopg for pgbouncer compatibility
database_url = "postgresql+psycopg://postgres:PASSWORD@db.PROJECT.supabase.co:5432/postgres"
engine = create_async_engine(
    database_url,
    poolclass=NullPool,  # pgbouncer handles pooling
)
```

#### TypeScript (Fastify)
```typescript
// Detects Supabase automatically
const isSupabase = databaseUrl.includes("supabase.com");
const poolConfig = {
  connectionString: databaseUrl,
  max: isSupabase ? 5 : 20,  // Lower for pgbouncer
};
```

#### Environment Variables
```bash
# Option 1: Direct connection string
DATABASE_URL=postgresql://postgres:PASSWORD@db.PROJECT.supabase.co:5432/postgres

# Option 2: Supabase credentials (auto-constructed)
SUPABASE_URL=https://PROJECT.supabase.co
SUPABASE_PASSWORD=your_password
```

## API Endpoints

- `POST /api/v1/workflows/generate` - Generate contract from NLP

## Security Considerations

### x402 Payments
- Always verify payment signatures server-side
- Use facilitator for payment settlement
- Implement spending controls (daily/monthly limits)
- Validate network support before processing payments

### Database (Supabase)
- Use connection pooling (pgbouncer) for production
- Never expose database credentials in frontend code
- Use environment variables for all secrets
- Enable SSL/TLS for database connections

### Smart Contracts
- Use OpenZeppelin libraries for security
- Audit contracts before mainnet deployment
- Verify contracts on block explorers
- Use Foundry/Hardhat for testing

### Network Security
- Validate chain IDs before processing transactions
- Use official RPC endpoints
- Implement rate limiting for API endpoints
- Monitor for suspicious activity

## Development Workflow

### Local Development
1. Start Supabase (or use cloud instance)
2. Configure `.env` with Supabase credentials
3. Run migrations: `alembic upgrade head`
4. Start Python backend: `uvicorn hyperagent.api.main:app --reload`
5. Start TypeScript API: `cd ts/api && npm run dev`
6. Start Next.js frontend: `cd frontend && npm run dev`

### Testing
- Foundry: `forge test`
- Hardhat: `npx hardhat test`
- Python: `pytest tests/`
- TypeScript: `npm test`

### Deployment
- Contracts: Deploy via Foundry/Hardhat scripts
- Backend: Deploy to cloud (Render, Railway, etc.)
- Frontend: Deploy to Vercel
- Database: Use Supabase cloud instance

## Code Structure

### Python Backend (`hyperagent/`)
- `api/` - FastAPI routes and middleware
- `core/` - Core services (generation, audit, deployment)
- `blockchain/` - Network configuration and blockchain operations
- `db/` - Database session management (Supabase)
- `api/middleware/x402.py` - x402 payment middleware

### TypeScript API (`ts/api/`)
- `src/routes/` - Fastify route handlers
- `src/x402/` - x402 payment verification client
- `src/storage/postgres.ts` - Supabase database store
- `src/evm/networkConfig.ts` - Network configuration
- `src/config/env.ts` - Environment configuration

### Frontend (`frontend/`)
- `app/` - Next.js App Router pages
- `components/` - React components
- `components/x402/` - x402 payment UI components

### Smart Contracts
- `contracts/` - Solidity contracts
- Uses OpenZeppelin libraries
- Compiled with Foundry/Hardhat
